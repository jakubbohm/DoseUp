Spike #50 — round-trip against a REAL Azure Service Bus BASIC namespace
=======================================================================
Run 2026-07-18 by Jakub (namespace) + Claude (smoke). This is the one thing the
emulator could not prove: that the queue-only design actually works on Basic tier.
No secrets in this file — the connection string never left the machine's scratchpad.

--- The namespace (Jakub, `az servicebus namespace show`) --------------------
  name:     doseup-spike-sb-basic
  sku/tier: Basic / Basic          <-- real Basic tier, not the emulator's Standard surface
  status:   Active
  location: germanywestcentral
  queues:   ["dose-events"]        <-- queues only; no topics/subscriptions exist

--- How it was run ----------------------------------------------------------
  PoC API (unchanged) pointed at the real namespace + a throwaway local Postgres:
    ConnectionStrings__messaging   = <RootManageSharedAccessKey conn string, Basic ns>
    ConnectionStrings__messagingdb = Host=localhost;Port=5433;Database=messagingdb;...
  Wolverine wiring is exactly the production-shaped config: AutoProvision OFF,
  SystemQueuesAreEnabled(false), explicit ListenTo/Publish on the `dose-events` queue.

--- Startup: connected to the real Basic queue, no system queues --------------

  grep -iE 'MessagingEntityNotFound|topic|subscription|session|not supported|

  grep -E '^(warn|fail|crit)'                                ->  (no matches)

--- Verdict ------------------------------------------------------------------
  #50 GO — now confirmed on a real ASB Basic namespace, not just the emulator.
  The queue-only, no-system-queues, AutoProvision-off design runs on Basic tier with
  a clean send+receive round-trip. Wolverine keeps its reply endpoint in-memory
  (stub://replies/), so it needs no ASB topic/subscription/session and no system queues
  — which is exactly why Basic tier is sufficient.

  Cleanup: local Postgres container + PoC process were torn down after the run.
  The `doseup-spike-sb-basic` namespace in resource group `doseup-spikes` was left
  running for Jakub to delete (his subscription).
